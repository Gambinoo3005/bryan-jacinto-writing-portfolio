<div class="company-background">
  <h2>Why I built it myself</h2>
  <p>Most writers hand you a PDF and a Google Drive folder. I wanted to show, not tell — so instead of dropping my work into template #47, I built the whole thing from scratch. The words are mine, the design is mine, and so is every line of code. The fact that you can read the source on GitHub is sort of the point: I can write the copy <em>and</em> ship the site it lives on.</p>
</div>

<div class="section-header">
  <h2>The stack</h2>
</div>

<p>It's a static <a href="https://astro.build" target="_blank" rel="noopener noreferrer">Astro</a> site styled with Tailwind CSS v4 and written in TypeScript. Content lives in MDX collections validated with Zod, so a missing field fails the build instead of shipping a broken page. Animations run on GSAP with Lenis for smooth scroll, and the whole thing deploys on Vercel. Almost nothing ships JavaScript by default — the interactive pieces load only when they're actually needed and motion is allowed.</p>

<div class="section-header">
  <h2>The parts I'm proud of</h2>
</div>

<p><strong>An experience timeline that behaves like a real deck of cards.</strong> Swipe it on a phone, click through it, or arrow-key it on desktop — it's a custom card-stack component with organic depth and physics, and it never hijacks the page scroll the way most carousels do.</p>

<p><strong>SEO wired the way I'd wire a client's.</strong> JSON-LD structured data (Person, Article, BlogPosting), a generated sitemap, canonical tags, and proper Open Graph and Twitter cards — not an afterthought, but the foundation.</p>

<p><strong>Accessible and fast on purpose.</strong> Semantic HTML, ARIA where it counts, full keyboard navigation, and reduced-motion support. Images are WebP, the hero is preloaded, and the theme switches with no flash of the wrong colors.</p>
